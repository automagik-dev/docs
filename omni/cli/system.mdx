# System Commands

Configuration, diagnostics, and service management. For detailed event, batch, and prompt references, see their dedicated pages.

## `omni config`

Manage CLI configuration — default instance, output format, API settings.

```bash
omni config list            # List all config values
omni config get <key>       # Get a value
omni config set <key> <val> # Set a value
omni config unset <key>     # Remove a value
```

Common settings:

```bash
omni config set instance <id>     # Set default instance
omni config set format json       # Default to JSON output
```

<Tip>
Run `omni config set instance <id>` to set a default instance. Most commands that accept `--instance` will use this default, saving you from passing it every time.
</Tip>

## `omni events`

Query the event log — message history, analytics, and replay. For full options, see the dedicated [Events & Analytics reference](/omni/cli/events).

```bash
omni events list --since 24h            # List recent events
omni events search <query>              # Search events by content
omni events analytics                   # Event analytics summary
omni events replay --start --since 24h  # Replay events
```

---

## `omni batch`

Batch processing for media transcription and text extraction at scale. For full options, see the dedicated [TTS, Batch & Prompts reference](/omni/cli/tts-batch-prompts).

```bash
omni batch estimate [options]           # Estimate scope and cost
omni batch create [options]             # Create a processing job
omni batch status <job-id> --watch      # Watch job progress
omni batch cancel <job-id>              # Cancel a running job
```

---

## `omni prompts`

Manage LLM prompt overrides for media processing. For full options, see the dedicated [TTS, Batch & Prompts reference](/omni/cli/tts-batch-prompts).

```bash
omni prompts list                       # List all prompts with override status
omni prompts get <name>                 # Show current prompt
omni prompts set <name> [value]         # Set override
omni prompts reset <name>               # Clear override, revert to default
```

Message journey tracing — track a message through the full Omni pipeline for latency debugging.

```bash

omni journey summary                    # Aggregated journey metrics
```

## Service Management

Start, stop, and restart the Omni server and NATS:

```bash
omni start      # Start Omni services (API + NATS)
omni stop       # Stop services
omni restart    # Restart services
omni status     # API health and connection info
omni update     # Update CLI to latest version
```

<Warning>
`omni stop` shuts down both the API server and NATS. All connected instances will disconnect and incoming messages will be queued until the service restarts. Use `omni restart` for zero-downtime config changes when possible.
</Warning>
