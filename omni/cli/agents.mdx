# Agent & Provider Commands

Hub page for AI agent infrastructure. Each area has a dedicated reference page with full options.

<Tip>
The typical setup flow is: create a **provider** (the AI backend), create an **agent** (bound to that provider), then create a **route** (which chats the agent handles). See [Agent Integration concepts](/omni/concepts/agents) for the full picture.
</Tip>

## `omni agents`

Manage AI agent entities.

```bash
omni agents list               # List all agents
omni agents get <id>           # Get agent details
omni agents create [options]   # Create a new agent
omni agents delete <id>        # Delete (soft-delete, sets inactive)
```

### Creating an Agent

```bash
omni agents create --name "Support Bot" --provider <provider-id>
```

<Note>
Deleting an agent is a soft-delete — it sets the agent as inactive. Routes pointing to an inactive agent will stop matching.
</Note>

## `omni providers`

Manage AI/agent providers — the backends that power agents. For full create/update options and setup wizards, see the dedicated [Providers reference](/omni/cli/providers).

```bash
omni providers list              # List available providers
omni providers get <id>          # Get provider details
omni providers create [options]  # Create a new provider
omni providers test <id>         # Test provider health
omni providers setup             # Interactive setup wizard
```

---

## `omni routes`

Manage agent routing rules. For full create/update options, see the dedicated [Routes, Persons & Auth reference](/omni/cli/routes-persons-auth).

```bash
omni routes list --instance <id>        # List routes for an instance
omni routes create [options]            # Create a new route
omni routes test [options]              # Test route resolution
omni routes metrics                     # View route cache metrics
```

---

## `omni keys`

Manage API keys for authenticating with the Omni API. For full options including scoped permissions, see [Routes, Persons & Auth](/omni/cli/routes-persons-auth#api-key-commands).

```bash
omni keys create --name "Production"    # Create a new API key
omni keys list                          # List all keys
omni keys revoke <id>                   # Revoke (disable) a key
```
