# /learn — Behavioral Correction

When a user corrects a mistake, `/learn` diagnoses which behavioral surface caused it and applies a minimal, targeted fix.

## When to Use

- User corrects agent behavior ("no, don't do that", "stop doing X")
- Agent made a mistake that should never recur
- User explicitly invokes `/learn`
- A pattern of repeated errors suggests a missing behavioral rule

## Flow

1. **Analyze the mistake** — what went wrong? Read conversation context, recent changes, relevant code.
2. **Determine root cause** — why did the agent behave this way? Missing rule? Stale convention? Wrong default?
3. **Diagnose the surface** — which behavioral surface needs to change?
4. **Propose minimal fix** — enter plan mode. Show exactly which file will change and why. One change per learning.
5. **Apply with approval** — user must approve before any write.
6. **Save to memory** — write the learning as a feedback memory in `.claude/memory/`.

## Writable Surfaces

| Surface | Path | What It Controls |
|---------|------|-----------------|
| Project conventions | `CLAUDE.md` | Commands, gotchas, project rules |
| Agent identity | `AGENTS.md` | Agent role, preferences, team behavior |
| Agent personality | `SOUL.md` / `IDENTITY.md` | Tone, communication style |
| Global rules | `~/.claude/rules/*.md` | Cross-project behavioral rules |
| Claude native memory | `.claude/memory/` | Feedback, user prefs, project context |
| Project memory | `memory/` | Project-scoped knowledge files |
| Hooks | `.claude/settings.json` | Event-driven automation |

## Never-Touch Surfaces

- `plugins/genie/skills/` — framework skills (maintained by framework developers)
- `plugins/genie/agents/` — framework agents
- Other projects' files
- Source code — learn updates behavior configuration, not implementation

## Memory Connection

Every learning is also saved as a feedback memory:

```markdown
---
name: <concise-name>
description: <one-line description>
type: feedback
---

<The rule itself>
**Why:** <reason or incident>
**How to apply:** <when/where this kicks in>
```

## Rules

- Plan mode is mandatory — never write without user approval
- One learning at a time
- Never assume — verify with the user before recording
- Never modify framework files
- Never write source code — behavioral configuration only
- Minimal changes — smallest rule that prevents recurrence
